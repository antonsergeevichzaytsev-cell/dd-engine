# FEL Methodology — Anton's Framework

**Источник опыта:** UMMC ~$550M gold-from-tailings (FEL 1-3 director), RUSAL CapEx portfolio ($350M+, 12 projects), ERG/McKinsey CapEx transformation engagements. Это не учебник по FEL — это рабочий стандарт по которому Anton ведёт защиту проектов.

**Назначение файла:** референс при DD взрослых проектов на этапе FEL 1-3, при оценке greenfield/expansion, при IC memo подготовке. Используется в любом sector skeleton при §10 (CapEx, FEL, Sustaining).

---

## §1. FEL базовая структура и accuracy targets

**FEL = Front-End Loading.** Industry-standard stage gating для capital projects. Этапы и точность:

| Stage | Назначение | Cost accuracy | Schedule accuracy | Engineering complete | Typical study cost |
|-------|------------|----------------|---------------------|------------------------|----------------------|
| FEL 0 (Identification) | Конъюнктура, ranging | ±50% | ±50% | <1% | <$0.5M |
| FEL 1 (Scoping/Conceptual) | Variant selection | ±35% | ±35% | 1-5% | $0.5-2M |
| FEL 2 (Pre-Feasibility/PFS) | Selected option developed | ±25% | ±25% | 5-15% | $2-8M |
| FEL 3 (Feasibility/BFS/FEED) | Sanction-ready | ±10-15% | ±10-15% | 15-30% | $5-25M |
| Detailed Engineering | Construction-ready (IFC) | ±5% | ±5% | 100% | Part of execution capex |

**AACE International cost estimate classes** mapping:
- AACE Class 5 ≈ FEL 0
- AACE Class 4 ≈ FEL 1
- AACE Class 3 ≈ FEL 2
- AACE Class 2 ≈ FEL 3
- AACE Class 1 ≈ Detailed Engineering

**Critical sanity check:** if stated accuracy doesn't match engineering completeness, the estimate is mis-classified. Common manipulation: presenting FEL 2 deliverables as FEL 3 to push faster sanction.

## §2. Anton's signature FEL deliverable checklist

Что должно быть в портфеле каждой FEL stage, чтобы пройти Anton's IC defence test.

### FEL 1 (Scoping)

**Обязательные deliverables:**
- [ ] Market study (preliminary): demand, supply, price outlook 3-5 years (CRU/Wood Mac/Argus/internal)
- [ ] Resource/feedstock confirmation (preliminary — JORC Inferred OK at this stage)
- [ ] Flowsheet selection rationale (3+ alternatives evaluated)
- [ ] Capacity range (±20% width acceptable)
- [ ] Location screening (3+ alternatives, qualitative)
- [ ] Concept-level layout
- [ ] ROM capex (factored from benchmark $/t)
- [ ] ROM opex (per established cost curves)
- [ ] Concept economics (NPV at expected price, ±35%)
- [ ] HSE concept (TSF, water, emissions concept)
- [ ] Stakeholder map (community, regulators, partners)
- [ ] Risk register (qualitative, top 15-20 risks)

**Common gaps in FEL 1 that I flag:**
- Single flowsheet "obviously best" — without alternatives evaluation
- Capex from one benchmark project (cherry-picked successful project)
- Resource based on historical drilling without QA/QC review
- No water/power availability check
- No community/Indigenous engagement plan
- Risk register copy-paste from prior project

### FEL 2 (Pre-Feasibility)

**Обязательные deliverables:**
- [ ] Confirmed market (off-take LOI or preliminary commitments)
- [ ] Resource at Indicated/Measured for >10 year LOM
- [ ] Locked-cycle test work for metallurgy (recovery confirmed)
- [ ] Detailed flowsheet with equipment sizing
- [ ] Plot plan and 3D layout (preliminary)
- [ ] Equipment list (major) with budget quotes from 2+ vendors
- [ ] Site investigation (geotech, hydro, environmental)
- [ ] Tailings location options analysis
- [ ] Power supply route firm (LOI or grid study)
- [ ] Water supply confirmed (legal source + rights)
- [ ] Logistics study (concentrate/product out, reagents in)
- [ ] Capex Class 3 with WBS and contingency built from risk register
- [ ] Opex Class 3 with detail by cost center
- [ ] Schedule Level 3 (master schedule, key milestones)
- [ ] Economic model (full DCF, sensitivities, Monte Carlo optional)
- [ ] Risk register quantitative (Probability x Impact for top 30 risks)
- [ ] ESG impact assessment (preliminary EIA, social baseline)
- [ ] Permitting roadmap with critical path
- [ ] Owner team plan and capacity assessment

**Common gaps in FEL 2:**
- Recovery from bench tests only (not locked-cycle) — typically fails on pilot
- Capex contingency arbitrary % (15% standard) without risk-based methodology
- Schedule "engineering-driven" not "execution-driven"
- Permitting timeline optimistic (regulator response time)
- No alternatives analysis on key decisions (TSF location, power source)
- Vendor budget quotes "verbal" or single-source

### FEL 3 (Feasibility/FEED)

**Обязательные deliverables:**
- [ ] Off-take agreements signed or final terms agreed
- [ ] Resource at Measured/Indicated, JORC/NI 43-101/equivalent technical report
- [ ] Pilot test work for any novel flowsheet element
- [ ] Full PFD, P&ID for all major systems
- [ ] Mass and energy balance signed
- [ ] Civil/structural/mechanical/piping/electrical/I&C engineering (Class 2)
- [ ] Equipment specifications, vendor pre-qualification, firm quotes for major
- [ ] Plot plan and 3D model (advanced)
- [ ] Construction execution plan (modules, sequencing)
- [ ] Construction contract strategy decided (EPC vs EPCM vs multi-contract)
- [ ] Schedule Level 4 (engineering-driven, by discipline)
- [ ] Capex Class 2 with quantified contingency
- [ ] Opex Class 2 with operator-level detail
- [ ] Economic model finalized (sensitivities, base/bull/bear)
- [ ] Risk register quantitative + mitigation cost
- [ ] EIA/ESIA completed (regulator-ready)
- [ ] All major permits secured or LOI from regulator
- [ ] Owner team mobilization plan (positions, hiring, contractor support)
- [ ] Financing arrangement (term sheet or commitment)
- [ ] Insurance arrangement
- [ ] Independent technical review (lender's engineer, IC TQR)

**Common gaps in FEL 3 that kill projects:**
- Off-take "in negotiation" — no signed terms
- Permits "expected" — not secured
- Power supply "letter of intent" only — not contracted PPA
- Contingency reverse-engineered to match economic model NPV target
- Schedule float removed to claim faster delivery
- Owner team plan not budgeted properly (typical underestimate)
- Insurance assumed without quotation

## §3. Defending capex at Investment Committee — Anton's framework

### 3.1 Independent cost estimation discipline

**Mandatory:** owner side runs parallel cost estimate to FEED contractor. NOT a check-the-box review — full bottoms-up estimate.

**How:**
1. Owner team builds own WBS (work breakdown structure)
2. For each major equipment item: 2-3 vendor quotes independently obtained
3. For each major construction package: published unit rates (RSMeans, AACE published) + market benchmarking
4. Bulks (concrete, steel, piping, electrical, instrumentation) calculated from quantities
5. Indirects (engineering, construction management, contractor overhead) at industry % bands
6. Contingency: risk-based, not percentage

**Owner estimate vs FEED estimate:**
- Within ±5%: estimates likely both robust, sanction
- 5-15% delta: investigate driver, reconcile
- >15% delta: red flag, deeper review needed

### 3.2 Contingency methodology — risk-based not percentage

**Wrong way (most common):**
- Contingency = 15% of total estimated cost. Done.

**Right way (Anton signature):**
1. Risk register quantitative: each risk has Cost Impact range (low/med/high) and Probability (low/med/high)
2. Monte Carlo simulation or analytical aggregation
3. Contingency = P50 contingency for "expected" sanction case, P75-P85 for protective sanction
4. Contingency reports separately for: scope uncertainty, estimating accuracy, market risk, schedule risk, execution risk

**Sanction discipline:**
- Don't approve project at P50 if any major risk has not been actively retired
- Specific reserves for known risks (e.g., schedule reserve for permit delay)
- Management reserve for unknown unknowns (typically 2-5% above contingency)

### 3.3 Schedule defence

**Common manipulations to flag:**
- Critical path artificially shortened by removing predecessor relationships
- "Fast track" claims without overlap of engineering and construction quantified
- Permitting compressed without regulator confirmation
- Commissioning duration unrealistic (always under-budgeted)
- Float removed from non-critical paths (creates fragile schedule)

**Anton's defence:**
- Schedule reverse-engineered from operating start date back: prove each milestone has at minimum industry-benchmark duration
- Construction productivity assumed: $/man-hour, m³ concrete/day, m piping/day — verify vs site/region productivity
- Commissioning sequence and duration: vendor input, operations input

### 3.4 Scope change discipline

**During execution (post-sanction), every scope change >$1M back to IC:**
- Driver: regulatory, market, technical, risk mitigation
- Cost impact (delta capex)
- Schedule impact (days/weeks delta)
- Resourced approval (project director + sponsoring exec + IC chair)

**Scope creep is the #1 killer of CapEx discipline.**

## §4. Specific patterns by sector

### 4.1 Aluminium smelter projects
- Power supply is single biggest risk variable
- Carbon supply (anodes) often under-engineered
- Casthouse capacity sizing often delayed
- HSE compliance (HF emissions, SPL) often under-budgeted

### 4.2 Copper concentrator projects (large porphyry)
- Water security: assumption stress test under climate change
- Tailings facility: often biggest single capex line
- Community/Indigenous: schedule risk if not advance
- Power supply: increasing requirement for renewable

### 4.3 Gold hydromet projects
- Refractory ore: pilot data non-negotiable
- Cyanide handling: regulatory and community risk
- Tailings: same general issues + cyanide destruction circuit
- Refining destination: critical for marketability (post-LBMA changes)

### 4.4 Nickel/PGM projects
- Cu-Ni sulphide concentrator: liberation studies (mineralogy)
- Smelter: SO₂ capture mandate is huge capex
- Refinery: technology lock-in (autoclave vs carbonyl vs EW)
- HPAL especially overrun-prone (historical record: 80-200% over original budget)

### 4.5 Alumina refinery
- Bauxite supply security long-term
- Red mud disposal capacity and method
- Energy efficiency lock-in (calciner type, heat integration)
- CBAM exposure for European-bound product

## §5. Owner team engineering — critical FEL success factor

See `methodology/owner_team_engineering.md` for full treatment.

**Headline:**
- Strong owner team during FEL = sanction at right capex, execution on plan
- Weak owner team = either fail to sanction (study budget burned) or sanction at inflated capex (overrun in execution)

**FEL phases require different owner team capabilities:**
- FEL 1: business + strategy heavy, light engineering
- FEL 2: balanced — engineering, commercial, operations early
- FEL 3: heavy engineering, contracting, construction prep
- Execution: PMO, contract management, operations interface

## §6. Common FEL failure modes — pattern library

### 6.1 "FEL gate skipping"
- Project pressured to compress FEL 1 → directly FEL 3 (or "FEL 2.5")
- Driver: management impatience, market window
- Outcome: cost overrun 30-80%, schedule overrun 50%+

### 6.2 "Vendor capture"
- Single vendor drove flowsheet selection during FEL 2
- FEL 3 estimates locked to that vendor pricing
- Alternative bidding becomes performative
- Outcome: 10-25% inflated capex

### 6.3 "FEED contractor capture"
- FEED contractor positions for execution EPC bid
- Engineering biased to favor their execution capability
- Capex underestimated to position for sanction (recoup in claims later)
- Outcome: change orders eat margin, owner pays

### 6.4 "Permit optimism"
- Schedule built on regulator providing approvals "as expected"
- Actual permitting takes 1.5-3x longer
- Outcome: schedule overrun, capital efficiency loss, sometimes deal-killer

### 6.5 "Owner team understaffed"
- Owner team during FEL = 20-30% of needed capability
- Outsourced to FEED contractor
- Owner can't challenge FEED outputs
- Outcome: contractor capture, capex inflation

### 6.6 "Contingency reverse-engineered"
- Economic model targets NPV at $X
- Contingency adjusted DOWN until NPV achieves $X
- Risk register exists but not driving contingency
- Outcome: actual capex hits real risks, no buffer

### 6.7 "Market timing optimism"
- Commodity price assumed at recent peak
- Decline scenarios not stress-tested
- Outcome: sanctioned project becomes underwater within 18 months of commissioning

## §7. FEL deliverable quality assessment framework

For each FEL deliverable, score:

| Quality dimension | Scoring guide | Red flag |
|---------------------|----------------|----------|
| Independence | 1 = single contractor; 5 = multiple independent parties | 1-2 |
| Bottom-up vs top-down | 1 = factored; 5 = quantity-take-off | 1-2 |
| Vintage of benchmarks | 1 = >5 years old; 5 = current | 1-2 |
| Test work depth | 1 = bench; 3 = locked-cycle; 5 = pilot/demo | 1-2 for FEL 3 |
| Regulatory engagement | 1 = none; 5 = LOI/permit secured | 1-2 for FEL 3 |
| Risk quantification | 1 = qualitative %; 5 = Monte Carlo with mitigation | 1-2 for FEL 3 |
| Owner team adequacy | 1 = severely under; 5 = best-in-class | 1-2 |
| Independent review | 1 = none; 5 = ITR completed | 1-3 for FEL 3 |

Total score <24 (out of 40) for FEL 3 = high probability of execution problems.

## §8. Application in DD context

### 8.1 DD on operating asset
- Look back at last 3 major capex projects: actual vs FEL 3 budget
- Pattern of overruns by stage
- Pattern by category (engineering, equipment, construction, indirects)
- Cause analysis (was contingency consumed by knowns or unknowns?)

### 8.2 DD on greenfield project pre-sanction
- Apply §7 framework
- Identify deliverable gaps
- Assess owner team capability
- Provide IC memo position: sanction-ready / further work required / do not proceed

### 8.3 DD on growth pipeline (operating asset with pipeline of projects)
- Same framework applied to each pipeline project
- Realistic phasing (can owner team handle multiple FEL 3s concurrent?)
- Aggregate capex vs balance sheet capacity

## §9. Anton's IC memo position language

Standard phrasing for IC memos (template for adaptation):

**For sanction-ready project:**
«Project [X] meets standards for FEL 3 sanction including: [list deliverables verified]. Capex estimate of $[Y]M ±[X]% is supported by [methodology]. Contingency of $[Z]M reflects quantified risks from risk register with P75 confidence. Execution plan accommodates [identified risks]. Recommendation: SANCTION.»

**For not sanction-ready:**
«Project [X] does not yet meet FEL 3 sanction standards. Specific gaps: [1, 2, 3]. Capex estimate likely understates true cost by [estimated %] based on [reasoning]. Required actions before sanction: [list]. Recommendation: DO NOT SANCTION at this gate. Return for IC review after [list of actions complete].»

**For do-not-proceed:**
«Project [X] economics do not support sanction at any contingency level. Drivers: [market, resource, capital cost, regulatory]. Recommendation: ABANDON or RE-SCOPE significantly.»
